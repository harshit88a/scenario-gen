 - TAB cycles display modes: behind-camera / top-camera / LiDAR BEV.
   For pedestrian scenarios use "behind"; for junction/multi-adversary
   use "lidar" BEV to see all actors relative to ego at once.

 - Set "loop": {"max_episodes": 5, "episode_duration_seconds": 20} in the
   generated JSON to auto-repeat the scenario and observe variance across
   different spawn rotations.

 - Add "display": {"camera": "top"} to the JSON to default to the elevated
   camera, which gives better situational overview for intersection scenarios.

 - For fog scenarios (T12) the LiDAR BEV is more informative than RGB, since
   the camera view will be visually washed out — exactly as intended.
